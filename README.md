# ロンテン(Long Teng)

**GitHub貢献者向けREADME**  
一般読者向けの説明は [note.md](note.md) をご覧ください。

---

このリポジトリは、真偽が不明なまま流通しやすく、論点が整理されないまま議論されがちな話題を、**可能な限り信頼できる一次情報から得た科学的な知見を基に整理する**ことを目的としています。

## プロジェクトの目標

健康・生活に関する話題について、**「何が確認されていて、何がまだわからないのか」を義務教育相当の知識で読めるMarkdown記事集を作る**こと。

## リポジトリ構成

```

├── motivation.md          # 管理者のモチベーション
└── topics/
    ├── TEMPLATE.md        # 新規トピック作成用テンプレート
```

## 基本方針（How）

1. **一次情報を優先する**: 公的機関の評価書・査読論文・系統的レビューを根拠の中核に置き、メーカー情報や個人ブログは補助にとどめる
2. **証拠の強さを明示する**: 「研究がある」「公的機関が整理している」「まだ不確実」を区別して書く
3. **両論を並べる**: 懸念を支持する根拠と、懐疑的・否定的な根拠の両方を省略せずに示す
4. **出典を必ずつける**: すべての主張に参考文献リンクを添える
5. **推論を見せる**: 「なぜその結論になったのか」の過程を、読者が追えるように書く
6. **断定を委譲する**: 最終的な判断は読者に委ね、「〇〇すべき」という結論の押しつけを避ける

詳細な執筆ガイドラインは [CONTRIBUTING.md](CONTRIBUTING.md) を参照してください。

## AIを活用した執筆プロセス

このリポジトリの記事は、**AI（大規模言語モデル）を活用**しています。透明性を重視し、そのプロセスを明示します。

### なぜAIを使うのか

人間が記事を書く場合、無意識のバイアス（先入観、都合の良い情報の選択、専門用語の使用など）が混入しがちです。AIを使う最大の理由は、**このような主観的なバイアスを徹底的に排除し、信頼できるソースに基づいた客観的な情報を、わかりやすくまとめる**ためです。

### ワークフロー

1. **AIによる情報収集と整理**
   - 公的機関の資料、査読論文、系統的レビューの検索と要約
   - 論点の抜け漏れチェック
   - 中学生でも理解できる平易な表現への翻訳
   - 論理構造の整理

2. **複数AIエージェントによる相互検証**
   - 異なるAIエージェントが独立してファクトチェック
   - 論理の飛躍や誤った推論の検出
   - バイアスの混入チェック

3. **人間による最終検証**
   - 引用元の原文との照合
   - 複数AIの出力を比較し、最終的な正当性を評価
   - 記事全体の方向性の判断

4. **コミュニティレビュー**
   - Pull Request、Issueでの継続的な改善

### 品質保証

- **AIによる客観化**: 人間の主観バイアスを排除
- **相互検証**: 複数AIエージェントによる多角的チェック
- **人間による最終責任**: 最終判断は人間が行う
- **オープンレビュー**: コミュニティによる継続的な改善

AIが書いたからといって信頼性が下がるわけではなく、むしろ**人間の主観を排除し、より客観的で正確な情報をまとめるツール**として活用しています。


## 参加方法

Issue や Pull Request での修正提案を歓迎します。以下のような貢献を特に歓迎します。

### 歓迎する貢献
- 📚 **文献の追加**: 公的機関の資料、査読付き論文、系統的レビューなど
- 🔍 **事実誤認の指摘**: 要約の誤り、データの読み違い、論理の飛躍など
- 📊 **根拠の強弱に関する補足**: この研究の限界、交絡要因の可能性など
- ⚖️ **利益相反の明確化**: 業界団体資料、スポンサー付き研究の位置づけ
- 🆕 **新規トピックの追加**: テンプレート（[topics/TEMPLATE.md](topics/TEMPLATE.md)）を参考に
- ✏️ **わかりやすさの改善**: より平易な表現、図解の追加、用語の解説など

### Pull Request を送る前に
1. [CONTRIBUTING.md](CONTRIBUTING.md) で執筆方針を確認してください
2. 追加する情報には、できるだけ一次論文、レビュー、公的機関文書へのリンクを添えてください
3. 「〇〇だから危険」「〇〇だから安全」という断定ではなく、「〇〇という証拠がある」「〇〇という限界がある」という書き方を心がけてください

### まずは Issue から
「こんな論点が気になる」「ここがわかりにくい」といったコメントも大歓迎です。
